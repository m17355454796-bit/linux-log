今天上午主要学习了命令行参数、环境变量以及程序地址空间的基础概念。

在 C 语言中，main 函数可以带参数，常见形式为：

int main(int argc, char* argv[])

其中，argc 表示命令行参数个数，argv 是字符串指针数组，保存每个命令行参数。运行程序时输入的内容，例如选项 -l、-a，都会作为参数传递给程序。Linux 中很多命令本质上都是可执行程序，例如 ls，程序会通过读取 argv 中的参数来决定执行不同功能。

环境变量本质上是一组键值对，常见的有：

PATH：系统查找可执行程序的路径列表
HOME：当前用户家目录
PWD：当前工作目录
OLDPWD：上一次工作目录

之所以输入 ls 就能直接运行，是因为 shell 会按照 PATH 中的路径依次查找对应程序；而自己写的程序通常要写成 ./prog，是因为当前目录默认不一定在 PATH 中。

还学习了 shell 内建命令和外部命令的区别。像 cd、export 这类命令属于 shell 内建命令，不需要单独依赖 PATH 查找外部程序。

另外，初步接触了程序地址空间，知道代码段、数据段、堆、栈、命令行参数和环境变量等内容。也学习了 static 和 const 的基本作用：static 可以让变量具有静态存储期，const 用于限制通过该标识符修改对象，提高代码安全性和规范性。然后const的作用是在编译时提醒程序员哪个不能修改的值被你修改了，不加const的话，这个值被修改了编译时不报错，运行时报错，比如char*ar=“aaaaa”；*a=‘a’；这个就是编译时不报错，运行时报错，以上就是我今天上午学习的内容
