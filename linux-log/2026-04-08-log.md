今天白天主要学习了三个内容，分别是 Makefile、Git，以及用 C 语言实现一个简单的进度条。

首先是 Makefile。今天主要理解了它的作用：当项目中不止一个源文件时，如果每次都手动输入 gcc 命令去分别编译、链接，会比较麻烦，这时候就可以把编译规则写进 Makefile 文件里，通过 make 来完成整个项目的构建。今天在 lesson1 和 lesson2 里都写了 Makefile，并且实际用 make 完成了编译，对“源文件先编译成 .o 文件，再链接生成可执行程序”这套流程有了更直观的认识。

其次是进度条的实现。今天先写了一个简单版本，后面又继续改成了一个可以传入当前进度和总量的版本。这个过程中把 process.h、process.c、main.c 分开写了，也结合 Makefile 进行了编译。进度条这部分主要用到了字符数组、百分比计算、\r 回到当前行开头、fflush(stdout) 立即刷新输出，以及 static 变量保存旋转符号状态。中间也出现了不少问题，比如格式控制符和变量类型对应不清、数组和数组元素混用、输出位置不对、主函数推进逻辑不够严谨、进度条宽度和终端窗口宽度不匹配等。后来经过反复修改，最终把进度条写到了可以正常运行的程度。

最后是 Git。今天重新看了一遍比特的 Git 课程，也结合白天自己的实际操作，对几个基本命令有了更清楚的认识。现在知道了 git add . 是把当前改动加入暂存区，git commit -m 是生成一次本地提交，git push 才是把本地提交上传到 GitHub，git status 可以查看当前仓库状态。.gitignore 则是用来忽略不需要上传的文件，比如 .o、.exe 这些编译产物。
